# 금융권 망분리 규정 (필수 준수)

본 문서는 모든 페르소나가 **절대 위반해서는 안 되는 망분리 규정**을 정리한다. Security 페르소나는 본 문서 위반 발견 시 Veto를 발동한다.

---

## 1. 망 구조

### 외부망 M365 테넌트
- 인터넷 통신 가능
- 외부 API, 웹사이트, 외부 AI 서비스 호출 가능
- **내부 시스템 직접 접근 불가**
- 개인정보·민감정보 저장·처리 **불가**

### 내부망 M365 테넌트
- 인터넷 통신 **차단**
- M365 내부 SaaS 서비스만 사용 가능 (Teams, SharePoint, Planner, Outlook, OneDrive, Forms 등)
- 외부 API/REST 호출 **차단**
- 모든 개인정보·민감정보 처리는 여기서만 수행

### 외부망 ↔ 내부망 연계
- **단방향(외부망 → 내부망)만** 원칙적으로 허용
- 승인된 게이트웨이/연계 채널을 통해서만 가능
- 양방향 또는 내부망 → 외부망 데이터 유출은 **절대 금지**

---

## 2. 데이터 분류 및 처리 위치

| 데이터 유형 | 외부망 | 내부망 |
|------------|--------|--------|
| 공개된 외부 데이터 (법령, 뉴스 등) | ✅ 처리·저장 | ✅ 처리·저장 |
| 회사 내부 일반 데이터 (일정, 업무 메모 등) | ⚠️ 원칙상 내부망 | ✅ 처리·저장 |
| **개인정보** (이름, 이메일, 사번, 연락처 등) | ❌ 저장·처리 금지 | ✅ 처리·저장 |
| **민감정보** (주민번호, 신용정보, 의료정보) | ❌ 절대 금지 | ✅ 처리·저장 (별도 통제 필요) |
| 인증 자격증명 (비밀번호, 토큰 등) | ⚠️ 외부망 자체 자격증명만 | ⚠️ 내부망 자체 자격증명만 |

---

## 3. 승인된 연계 방식

### 3.1 일방향 데이터 전송
- 외부망에서 수집한 비민감 데이터를 내부망으로 일방향 전송
- 예: 외부망에서 법령 변경 정보 수집 → 텍스트만 내부망 SharePoint 리스트에 적재
- 전송 채널: 회사 승인 게이트웨이 (담당 부서 사전 확인 필수)

### 3.2 알림성 메시지 전송
- 외부망에서 발생한 이벤트를 내부망 사용자에게 알림
- 예: 외부망 Power Automate가 내부망 Teams로 알림 카드 전송
- **알림 본문에 개인정보 미포함**이어야 함

### 3.3 금지되는 연계
- ❌ 내부망 데이터를 외부망으로 전송
- ❌ 외부망과 내부망의 양방향 동기화
- ❌ 외부 API에 내부 사용자 인증정보 전달
- ❌ 외부 AI 모델에 내부 문서 원문 전송 (요약·분석 목적 포함)

---

## 4. 위반 사례 (실제 발생 가능 시나리오)

### 4.1 잘못된 설계 — 외부망에서 직원 정보 처리
```
외부 법령 API → 외부망 Power Automate → 직원별 알림 메일 발송
                                          ↑ 직원 이메일 주소 = 개인정보
```
**문제**: 외부망 Power Automate가 직원 이메일 주소를 가지고 있음 → 개인정보 외부망 저장 위반

**올바른 설계**:
```
외부 법령 API → 외부망 Power Automate (법령 텍스트만 추출)
              → 게이트웨이 → 내부망 SharePoint (텍스트 적재)
              → 내부망 Power Automate → Teams 알림 (직원 정보는 내부망에만)
```

### 4.2 잘못된 설계 — 외부 AI에 내부 문서 전송
```
내부망 사용자 → Copilot Studio → AI Builder (외부 OpenAI) → 요약 결과
               ↑ 내부 문서 원문 전송
```
**문제**: 내부 문서가 외부 AI 모델로 전송됨 → 망분리 위반

**올바른 설계**:
- 회사 승인 내부 AI 모델만 사용
- 또는 외부 AI 사용 시 회사 승인 + 데이터 분류 검토 완료된 케이스만

---

## 5. 페르소나별 행동 요약

- **Architect**: 위 규정을 1차 배치 결정의 절대 기준으로 적용
- **Developer**: 내부망 플로우에서 인터넷 커넥터(HTTP, 외부 API) 사용 금지
- **Security**: 본 문서 위반 발견 시 즉시 Veto
- **Documentation**: 설계서에 본 규정 적용 결과를 명시
- **Orchestrator**: 사용자 요구가 본 규정과 충돌하면 요구 축소 또는 대안 협상 주도
