# 개인정보보호 및 컴플라이언스

금융권 회사로서 적용받는 법령·규제와 그에 따른 설계 시 준수사항.

---

## 1. 적용 법령

| 법령/규제 | 적용 범위 | 핵심 요구사항 |
|---------|-----------|--------------|
| 개인정보보호법(PIPA) | 모든 개인정보 처리 | 수집 목적 명시, 동의, 최소수집, 안전조치, 파기 |
| 신용정보법 | 신용정보 처리 | 신용정보 별도 통제, 외부 전송 제한 |
| 전자금융거래법 | 전자금융 시스템 | 망분리, 안전성 기준 준수 |
| 금융감독원 전자금융감독규정 | 금융 IT 운영 | 망분리 의무 (외부 인터넷 구간 vs 업무망) |
| 정보통신망법 | 정보처리 시스템 일반 | 접근통제, 로깅 |

---

## 2. 개인정보 정의 (본 시스템 기준)

다음을 개인정보로 간주한다:

### 2.1 직접 식별 가능한 정보
- 성명, 사번
- 이메일 주소 (회사 또는 개인)
- 전화번호
- 주민등록번호, 운전면허번호
- 사진, 음성, 영상

### 2.2 결합 시 식별 가능한 정보
- 부서 + 직급 + 입사년도 (조합 시 특정 가능)
- 지점명 + 직무 (소규모 지점의 경우)

### 2.3 민감정보
- 신용정보 (대출, 카드 사용 내역 등)
- 의료정보
- 정치 성향, 종교, 노조 가입 등

## 3. 설계 시 준수사항

### 3.1 수집·이용
- **목적 명시**: 설계서에 "이 자동화가 수집·이용하는 개인정보"를 명시
- **최소 수집**: 자동화에 꼭 필요한 항목만 처리 (예: 이름만 필요한데 사번까지 가져오지 말 것)
- **외부 동의 불요 케이스 확인**: 사내 업무용 자동화는 일반적으로 "사용자와의 계약 이행"으로 동의 불요지만, 별도 처리 시 검토

### 3.2 저장
- 개인정보는 **내부망에만 저장**
- 외부망 SharePoint·OneDrive에 개인정보 저장 절대 금지
- 보존기간 명시 (예: 알림 로그 90일 후 자동 삭제)

### 3.3 전송
- 외부 API에 개인정보 전송 금지
- 내부망 내부 전송은 허용 (Teams, Outlook 등)
- 외부 알림 메시지에 개인정보 포함 금지 (예: "홍길동님의 휴가 신청" → "휴가 신청건 1건")

### 3.4 파기
- 보존기간 경과 시 자동 삭제 메커니즘 설계
- 수동 파기 절차 명시 (담당자, 주기)

### 3.5 접근 통제
- 자동화가 처리하는 개인정보는 **최소 권한 원칙** 적용
- 서비스 계정 권한 명시 (예: 특정 SharePoint 리스트 읽기 권한만)
- 감사 로그 활성화

## 4. 자주 발생하는 컴플라이언스 위반

### 4.1 "내부 업무니까 괜찮아" 오류
잘못된 가정: "사내 업무 자동화니까 직원 정보 외부망 처리 OK"  
실제: 개인정보보호법은 처리 주체와 무관하게 적용. 직원 정보도 개인정보.

### 4.2 "마스킹했으니 괜찮아" 오류
잘못된 가정: "이름 첫 글자만 표시했으니 개인정보 아님"  
실제: 마스킹 수준에 따라 다름. 결합 시 식별 가능하면 여전히 개인정보. 최소 부서·직급 조합도 위험.

### 4.3 보존기간 미설정
잘못된 설계: 알림 로그를 무기한 SharePoint에 적재  
올바른 설계: 90일 후 자동 삭제 플로우 별도 운영

### 4.4 외부 AI에 내부 문서 학습
AI Builder 외부 모델, 외부 OpenAI 등에 내부 문서를 보내면 학습 데이터에 포함될 수 있음. 회사 승인 모델만 사용.

## 5. 페르소나별 적용

- **Architect**: 데이터 종류 식별 시 본 문서의 개인정보 정의를 기준으로 분류
- **Developer**: 플로우에 개인정보 처리 단계 있으면 최소 수집·보존기간 설계 명시
- **Security**: 본 문서 항목을 체크리스트로 변환해 검토
- **Documentation**: 설계서 "개인정보 처리 항목" 절을 별도로 명시
