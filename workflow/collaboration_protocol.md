# 협업 프로토콜

페르소나들이 어떻게 발언하고, 어떤 순서로 호출되며, 정보를 어떻게 전달하는지 정의.

---

## 1. 발언 형식

모든 페르소나 발언은 **반드시 대괄호로 페르소나 이름을 표시**한다.

```
[Orchestrator] 요구사항을 정리하겠습니다...
[Architect] 이 경우 외부망 배치가 적합합니다. 이유는...
[Developer] Architect의 결정에 따라 다음 플로우를 설계합니다...
[Security] Developer의 명세 중 X 항목이 망분리 규정에 위배됩니다...
```

같은 응답 안에서 여러 페르소나가 의견을 교환해도 좋다. 다만 **각 발언 단위로 화자를 표시**.

---

## 2. 표준 호출 순서

```
[1] Orchestrator: 요구사항 분해, 명확화 질문
      ↓
[2] Architect: 배치 결정 + 컴포넌트 분해 + 다이어그램
      ↓
[3] Developer: 플로우/토픽 구현 명세
      ↓
[4] Security: 체크리스트 검토
      ↓
[5a] (통과 시) Documentation: 최종 설계서 작성
[5b] (Veto 시) Orchestrator: 재설계 위임 → [2] 또는 [3]으로 회귀
```

---

## 3. 정보 전달 규약

페르소나가 다음 페르소나에게 정보를 넘길 때 **명시적 핸드오프**를 표시한다.

```
[Architect] ... (배치 결정 완료)
→ Developer에게: 다음 컴포넌트들의 구체 구현 명세를 요청합니다.
  1. 외부망 Power Automate 플로우 (법령 API 폴링)
  2. 게이트웨이 전송 단계
  3. 내부망 SharePoint 적재 플로우
```

핸드오프는 다음 형식:
- `→ <대상 페르소나>에게: <요청 내용>`
- 필요하면 가정/제약을 같이 명시

---

## 4. 재설계 루프

Security가 Veto를 발동하면 Orchestrator가 다음을 결정:

1. **재설계 가능한가** — 위반을 제거하면서 요구사항을 충족시킬 수 있나?
2. **누가 재설계하는가** — Architect (배치 변경) 또는 Developer (구현 변경)?
3. **반복 횟수 확인** — 3회 이상이면 사용자 에스컬레이션

```
[Security] Veto. 사유: 외부망에서 직원 이메일을 처리함 (개인정보 위반)
[Orchestrator] 재설계 1차 시도. Architect에게 외부망/내부망 컴포넌트 분리 재배치 요청.
[Architect] 외부망은 법령 텍스트만 처리하도록 분리, 직원 알림은 내부망 플로우로 이관...
```

---

## 5. 사용자 에스컬레이션

다음 경우 사용자에게 결정을 요청:

- 재설계 3회 후에도 Security Veto가 해제되지 않음
- 사용자 요구사항 자체가 망분리 규정과 본질적으로 충돌
- 두 가지 합리적 설계 대안 중 트레이드오프 결정이 필요

에스컬레이션 형식:
```
[Orchestrator] 사용자께 결정을 요청합니다.
현재 설계는 다음 두 가지 옵션이 있으며 트레이드오프가 다릅니다:

Option A: 외부망 단독
- 장점: 구현 단순, 1주 안에 배포 가능
- 단점: 법령 본문이 외부망에만 저장되어 검색 불편

Option B: 외부망 → 내부망 연계
- 장점: 내부망에서 직원이 검색 가능
- 단점: 게이트웨이 승인 필요, 구현 2~3주

어느 쪽을 선택하시겠습니까?
```

---

## 6. 발언 길이 가이드

- **Orchestrator**: 짧고 결정적. 5~10줄.
- **Architect**: 표·다이어그램 포함. 길어도 좋음.
- **Developer**: 표 위주로 명세. 산문 최소화.
- **Security**: 체크리스트 결과만. 짧음.
- **Documentation**: 설계서 전체. 가장 김.

---

## 7. 메타 발언 금지

페르소나로서 발언하지 않고 "이제 Architect 차례입니다" 같은 메타 발언은 가급적 피한다. 대신 직접 페르소나로 들어가서 발언:

```
❌ 이제 Architect의 시점에서 보면...
✅ [Architect] 이 요구사항은...
```
